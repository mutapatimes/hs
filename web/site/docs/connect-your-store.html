Docs/Step 1

Connect your store.

Connecting lets Halia read who has bought from you, so it can find your quiet big spenders. It's read-only, Halia can look, but never change anything, and it never keeps a copy of your customers. Pick your platform below.

Before you start

You'll need to be signed in to your store's admin. That's it, no developer, no code.

Halia connects to one store per account. If you run more than one shop, connect each as its own account. Whichever platform you're on, the connection is read-only: Halia reads your customers and orders to score them, and nothing more.

Shopify

SShopifyEasiest

If you're on Shopify, this takes about two minutes, there are no keys to copy.

  1. Start the connection

    Go to haliascore.com/connect and type your store address (it ends in .myshopify.com).

  2. Approve read-only access

    Shopify shows you exactly what Halia can see, your customers and orders, and asks you to approve. Click approve.

  3. You're done

    Halia starts finding your VICs. Your dashboard is usually ready within the hour.

Halia can only look. It reads your orders to score them, it can never change a thing or place an order. (You may notice it asks to add tags too; that only ever happens later, when you ask it to label a customer for outreach.)

WooCommerce

WWooCommerceA few minutes

WooCommerce needs a read-only “key”, think of it as a look-but-don't-touch password you create for Halia.

  1. Open your API keys

    In your WordPress admin, go to WooCommerce → Settings → Advanced → REST API.

  2. Add a key for Halia

    Click Add key, name it “Halia”, and set the permission to Read. Save.

  3. Copy the two codes

    WooCommerce shows a key and a secret. Copy both, they're only shown once.

  4. Paste them into Halia

    Go to haliascore.com/connect, choose WooCommerce, and paste your store address and the two codes.

One thing to check: your store needs to run on a secure (https) address. Almost every store already does.

BigCommerce

BBigCommerceA few minutes

BigCommerce needs a read-only token, a code you create that lets Halia look at your orders.

  1. Create an API account

    In your admin, go to Settings → API → Store-level API accounts and click Create.

  2. Set it to read-only

    Set Orders and Customers to read-only, and leave everything else off. Save.

  3. Copy your token and store ID

    Copy the token it gives you. Your store ID is the short code in your admin web address (store-XXXX).

  4. Paste them into Halia

    Go to haliascore.com/connect, choose BigCommerce, and paste them in.

What happens next

Halia scores your customers and opens your dashboard, your most valuable clients at the top, everyone graded, and the reason behind each one.

For a big back-catalogue the first run can take up to an hour; you'll get an email when it's ready. After that, every new order is checked the moment it comes in. Next, learn how to use your dashboard.

If something's off

What you're seeingWhat to do
No customers showed upDouble-check the key was set to Read, and that your store has past orders.
It says it couldn't connectCreate a fresh key or token and paste the new one. Make sure your store address is exact.
Still stuckEmail hello@haliascore.com and we'll connect it with you, happy to hop on a call.
