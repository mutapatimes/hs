Connect your store.

Halia connects to one storefront at a time, read-only. It can read your customers and orders to score them — it can never change, delete, or place an order. Pick your platform below.

On this page

Overview

Every connection Halia makes to a store is read-only and scoped to the minimum it needs: your customers and their order history. Nothing is written back to your storefront unless you explicitly trigger it (for example, applying a Halia:A tag or creating a segment on Shopify). Customer data is scored in memory and discarded within minutes — Halia never stores a copy.

You connect one store per account. If you run several storefronts, connect them as separate accounts. Ready when you are — start at the connect wizard, which detects your platform automatically.

Shopify

The fastest path: install the Halia app from your admin and approve read access. No keys to copy.

ShopifyOne-click install
  1. Open the connect wizard

    Go to halia.app/connect and enter your your-store.myshopify.com domain.

  2. Approve read access

    Shopify shows the permissions Halia requests — read_customers and read_orders. Approve to continue.

  3. Halia scores your store

    Your customers are pulled and graded. For a large back-catalogue the first run can take up to an hour; you'll be emailed when it's ready.

  4. Open your dashboard

    Your potential VICs appear, ranked, with the reasons behind every score.

Why the write permission? Halia only ever reads to score. A separate write_customers permission is used solely when you ask Halia to tag a customer or build a Shopify segment for outreach — never automatically.

WooCommerce

WooCommerce uses a read-only REST API key you create in your WordPress admin.

WooCommerceRead-only API key
  1. Open the REST API settings

    In your WordPress admin, go to WooCommerce → Settings → Advanced → REST API.

  2. Add a key

    Click Add key, give it a description like “Halia”, and set Permissions to Read.

  3. Copy the credentials

    Copy the Consumer key and Consumer secret shown once. Keep them safe.

  4. Paste them into Halia

    At halia.app/connect, choose WooCommerce and paste your store URL, consumer key, and secret.

Requirements. Your store must be reachable over HTTPS and have the WooCommerce REST API enabled (it is by default). Read permission is all Halia needs.

BigCommerce

BigCommerce uses a store API account with read-only scopes. You'll paste a store hash and access token.

BigCommerceStore-hash + token
  1. Create an API account

    In your admin, go to Settings → API → Store-level API accounts and click Create API account.

  2. Grant read-only scopes

    Set Orders and Customers to read-only. No other scopes are needed.

  3. Copy the token and store hash

    Save the Access Token. Your store hash is the code in your admin URL: store-{hash}.mybigcommerce.com.

  4. Paste them into Halia

    At halia.app/connect, choose BigCommerce and paste the store hash and access token.

After you connect

Halia scores every customer and opens your dashboard with your potential VICs surfaced and ranked A* to C. From there you can build segments, tag profiles, and draft outreach — which you send from your own tools.

Once connected, Halia also scores new orders in real time: the moment a VIC-grade order lands, you can get a web-push and email alert (and a Slack message, if you've connected it). See CRM & email connections to send grades onward.

Troubleshooting

SymptomFix
No customers appear after connectingCheck the API key has Read permission for orders and customers, and that the store has order history.
“Could not authenticate” / 401Regenerate the key (WooCommerce) or access token (BigCommerce) and paste the new value. Confirm the store URL / hash is exact.
Shopify install won't completeMake sure you're signed in to the correct store admin, then reopen /connect with your .myshopify.com domain.
Still stuckEmail hello@halia.app — we'll connect it with you.
